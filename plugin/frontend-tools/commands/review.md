对指定的前端代码文件进行系统性代码审查，输出结构化审查报告。

## 审查维度

### 1. 可访问性（a11y）
- 语义化 HTML、ARIA 使用是否合理
- 键盘导航与焦点管理
- 色彩对比与屏幕阅读器友好
- 表单标签与错误提示是否关联

### 2. 性能
- 包体与依赖是否合理（有无巨型依赖、重复打包）
- 路由与组件懒加载是否使用
- 图片/资源是否优化（尺寸、格式、懒加载）
- Core Web Vitals 相关（LCP、FID、CLS）是否存在明显问题
- 是否避免内联大对象、频繁重渲染

### 3. React / Vue 最佳实践
- React：Hooks 规则、依赖数组、避免不必要的 re-render、key 使用
- Vue：Composition API 使用、响应式陷阱、v-for key
- 单文件组件结构、props 类型定义、状态提升是否合理

### 4. 安全性
- XSS：是否对用户输入做转义/ sanitize，dangerouslySetInnerHTML 等使用是否必要且安全
- 依赖安全：已知漏洞依赖、锁文件是否提交
- 敏感信息是否硬编码（API Key、token）

### 5. 代码质量
- 命名是否符合项目规范（camelCase/PascalCase/kebab-case）
- 函数/组件职责是否单一
- 错误处理与边界情况
- TypeScript：是否避免 any、类型是否完整
- 代码风格与 rules 中的 coding-style 一致

## 输出格式

对每个发现的问题，按以下格式输出：

**[严重程度] 问题描述**
- 位置：`文件名:行号`
- 问题：具体说明
- 建议：如何修复（可附简短示例）

严重程度分级：🔴 必须修复 | 🟡 建议修复 | 🔵 可以优化

最后给出总体评价和 1–3 条优先修复建议。
