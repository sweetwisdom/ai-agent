在实现任何代码修改之前，先输出一份结构化计划，等待用户确认后再开始执行。

## 计划格式

请按以下格式输出计划：

### 任务理解
用 1-2 句话复述你理解的任务目标。

### 影响范围分析
列出将要修改的文件及修改原因：
- `文件路径`：修改原因

### 实现步骤
按顺序列出具体步骤，每步说明做什么、为什么这样做。

### 风险点
列出可能的风险或需要特别注意的地方（类型安全、依赖升级、破坏性变更、可访问性等）。

### 验收方式
说明如何验证修改正确：
- 构建命令：`pnpm build` 或 `npm run build`
- 校验命令：`pnpm lint` 或 `npm run lint`
- 测试命令：`pnpm test` 或 `npm run test`
- 手动验证步骤（如有）

---

输出计划后，**停止等待用户确认**，不要自动开始实现。
