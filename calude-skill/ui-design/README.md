## 🎯 简介

专业的全栈工程师级别 UI/UX 设计助手，支持**多平台**（PC 桌面端、移动端、平板、响应式 Web）的完整设计与原型实现。AI 会智能分析用户需求，自动识别目标平台，从产品需求分析、信息架构设计、用户体验优化到高保真 HTML 原型生成，提供一站式的跨平台设计解决方案。

### 为什么选择这个 Skill？

- ✅ **全流程自动化**：从需求分析到代码实现一站式完成
- ✅ **专业级设计**：符合现代 Web 设计规范和最佳实践
- ✅ **高质感界面**：玻璃拟态、渐变背景等现代视觉效果
- ✅ **即时可用**：生成的 HTML 原型可直接在浏览器打开
- ✅ **真实资源**：使用真实图片，非占位符
- ✅ **结构清晰**：多页面独立管理，iframe 统一展示

## ❓ 常见问题

### Q1: 生成的代码可以直接用于生产环境吗？

**A:** 这是高保真原型，适合用于：
- ✅ 产品演示和展示
- ✅ 前期开发参考
- ✅ 设计评审
- ❌ 直接生产环境（需要进一步工程化）

生产环境建议：
- 组件化重构
- 添加状态管理
- API 接口对接
- 性能优化
- 安全加固

### Q2: 为什么使用 iframe 平铺方式？

**A:** 为了方便一次性展示所有页面设计，无需点击跳转即可浏览全部界面。这种方式适合：
- 设计评审会议
- 产品演示
- 快速预览所有页面

如需传统导航方式，可以告知 AI 调整。

### Q3: 如何处理大代码量文件？

**A:** AI 会自动检测代码量：
- 单文件 < 500 行：一次性生成
- 单文件 > 500 行：分次写入
  1. 先创建文件框架
  2. 分批填充内容
  3. 每次告知进度

### Q4: 可以指定使用其他 CSS 框架吗？

**A:** 默认使用 Tailwind CSS，如需其他框架：
- Bootstrap：在需求中说明
- Material UI：在需求中说明
- 自定义：提供具体要求

### Q5: 生成的图片是真实的吗？

**A:** 是的，使用真实图片来源：
- Unsplash API（高质量免费图片）
- Pexels API（专业图片库）
- 根据场景自动选择合适的图片

### Q6: 如何避免使用 list-item 类名？

**A:** Skill 已内置规则，自动避免使用 Tailwind 的 `list-item` 类名，改用其他方式实现列表样式。

### Q7: 滚动条如何隐藏？

**A:** 自动添加 CSS 规则：
```css
.scrollable {
    overflow-y: auto;
    scrollbar-width: none; /* Firefox */
    -ms-overflow-style: none; /* IE 10+ */
}
.scrollable::-webkit-scrollbar {
    display: none; /* Chrome, Safari, Edge */
}
```

### Q8: 支持响应式设计吗？

**A:** 是的，生成的代码包含响应式布局：
- 使用 Tailwind 的响应式类（sm、md、lg、xl）
- 自适应不同屏幕尺寸
- 移动端友好

### Q9: 如何定制设计风格？

**A:** 在需求中说明设计偏好：
```
设计要求：
- 色彩：蓝色系，科技感
- 风格：现代、简约
- 特效：使用玻璃拟态和渐变
- 主题：支持暗色模式
```

### Q10: 生成后可以修改吗？

**A:** 完全可以！生成的代码：
- 结构清晰，注释充分
- 易于理解和修改
- 可以直接编辑 HTML 文件
- 可以调整 Tailwind 类名

## 📝 注意事项

### 技术限制

- ⚠️ 禁止使用 Tailwind 的 `list-item` 类名
- ⚠️ iframe 嵌入可能影响部分交互（如全屏模态框）
- ⚠️ CDN 资源需要网络连接
- ⚠️ 大文件会分次写入

### 最佳实践

- ✅ 优先使用 Tailwind 原子类
- ✅ 图标优先使用 FontAwesome
- ✅ 代码添加适当注释
- ✅ 保持文件结构清晰
- ✅ 测试不同分辨率
