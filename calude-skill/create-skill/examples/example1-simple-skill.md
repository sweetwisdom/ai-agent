# 示例1：创建简单的代码审查Skill

## 场景描述
用户想创建一个简单的代码审查skill，只需要Markdown提示词，不需要额外脚本。

## 使用过程

### 用户输入的需求
```
需求：我想创建一个代码审查skill
功能：
- 自动检查代码质量
- 提供改进建议
- 检查安全问题
输入：用户提供要审查的代码文件路径
输出：结构化的审查报告
```

### 系统引导问题与回答

**问题1：Skill基本信息**
- Skill名称：`code-review-skill`
- Skill描述：智能代码审查助手，提供代码质量、性能和安全方面的专业建议
- 主要用途：帮助开发者发现代码问题并提供改进建议

**问题2：功能详情**
- 核心功能：
  1. 代码质量检查（命名规范、代码结构）
  2. 性能优化建议
  3. 安全漏洞检测
  4. 最佳实践建议
- 输入要求：代码文件路径或代码片段
- 输出格式：分类的审查报告（问题严重程度、具体建议）

**问题3：实现方式**
- ✅ 纯Markdown提示词

**问题4：附加选项**
- ✅ 需要用户交互确认
- ✅ 需要示例文件
- ❌ 不需要配置文件
- ✅ 需要错误处理

### 生成的文件结构

```
code-review-skill/
├── skill.md           # 核心skill定义（包含详细的审查提示词）
├── examples/
│   ├── example1.md    # Python代码审查示例
│   └── example2.md    # JavaScript代码审查示例
└── README.md          # 使用文档
```

## 关键要点

1. **纯Markdown适用场景**：不需要执行系统命令或复杂逻辑处理
2. **提示词设计**：详细说明审查维度和输出格式
3. **示例很重要**：帮助用户理解如何使用
4. **中文文档**：所有内容使用中文，便于理解

## 预期效果

生成的skill可以直接使用，用户只需：
1. 将skill移动到 `.claude/skills/` 目录
2. 调用时提供代码文件路径
3. 获得结构化的审查报告
