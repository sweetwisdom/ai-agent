# UI/UX 描述提示词生成

将用户简短的产品想法转化为**结构化的 UI/UX 描述提示词**，便于交给设计工具、设计师或作为前端/产品需求与后续 AI 生成界面的输入。

## 从输入中识别

- **产品类型**：音乐、电商、社交、工具、内容、教育等
- **风格/参考**：如 Apple Music、某品牌、极简、拟物等
- **核心能力**：播放、购买、聊天、记录、搜索等

根据类型与风格推断合理的全局视觉与页面划分，不必写死某一类产品。

## 输出结构（Markdown）

必须包含以下部分，使用以下模板：

```markdown
创建完整的[风格/类型][产品类型]App。
全局风格： [3～6 个关键词或短语，如毛玻璃效果、圆角、深色模式、主色调]
需要以下页面/模块：
【页面/模块名】 功能点1、功能点2、功能点3
【页面/模块名】 功能点1、功能点2
…
【迷你/常驻组件】（若有） 功能点
```

- **总述句**：一句完整总述（「创建完整的 XXX 风格/类型 App」）。
- **全局风格**：3～6 个关键词或短语，用顿号或逗号分隔。
- **页面/模块**：每行 `【模块名】` 后跟该模块下的功能点（简短词组）。
- **迷你/常驻组件**：如底部迷你播放条、悬浮按钮等，可单独一行。

## 示例

**输入：**
```
我想做一个Apple Music风格音乐播放器App
```

**输出：**
```markdown
创建完整的Apple Music风格音乐播放器App。
全局风格：毛玻璃效果、圆角、深色模式、红色主调
需要以下页面/模块：
【首页】推荐歌单、新歌速递、排行榜入口
【播放器】专辑封面、同步歌词、进度条、播放控制、音量
【播放列表】当前队列、拖拽排序、删除
【搜索】搜索框、热搜、历史、结果分类（歌曲/专辑/艺人）
【个人库】最近播放、收藏、下载、自建歌单
【歌单详情】封面、歌曲列表、播放全部、收藏
【艺人页】头像、热门歌曲、专辑列表、简介
【设置】音质、均衡器、下载、账户、通知开关
【迷你播放条】底部常驻、封面+标题+播放控制
```

**输入：**
```
做一个极简待办事项工具，参考 Things 风格
```

**输出：**
```markdown
创建完整的极简 Things 风格待办事项App。
全局风格：圆角、浅色主调、大留白、清单式布局、柔和阴影
需要以下页面/模块：
【今日】今日任务列表、添加、完成/推迟
【清单】多清单、清单内任务、拖拽排序
【任务详情】标题、备注、截止日、子任务、标签
【添加/编辑】标题、日期与时间、提醒、清单选择
【设置】主题、通知、同步、数据
```

## 注意

- 功能点用简短词组，同一模块内用顿号或逗号分隔。
- 模块数量与粒度随产品类型调整（音乐类可多页面，工具类可更少）。
- 更多示例见 [examples.md](examples.md)。
